## Workflow Patterns

### Code Work
1. Open project in IDE
2. Read `CLAUDE.md` and `docs/lessons/` first
3. Make changes
4. Verify: `tsc --noEmit` → `npm test` → `npm run build`
5. Commit and push to `main` → Vercel auto-deploys

### Content Work
1. Draft in markdown (content repo)
2. Git commit + push
3. Seed into Sanity CMS (for website content)
4. Or copy-paste to social media platforms

### Scouting Work (Before Maps specific)
1. Travel to location
2. Document in field notes (markdown)
3. Add to Uddip Content → `before-maps/` folder
4. Create Sanity field note document
5. Update scouting log on website

## Decision-Making Framework

1. **Does it compound?** — Prefer work that builds on itself
2. **Ship it** — Imperfect and live beats perfect and stuck
3. **Document everything** — Future you (or future AI) needs context
4. **Fun-max** — If it's not fun, redesign the system
